# Изоляция клиентов в Unisender Go: Project на клиента

Зачем: у Unisender Go suppression-лист (жалобы/отписки) и репутация отправки
общие на аккаунт, если не разносить клиентов по отдельным Project'ам.
Один клиент с плохой базой/жалобами может задеть доставляемость остальных.
Project даёт каждому клиенту свой домен, свой suppression-лист и свой API-ключ.

Создание Project и добавление домена через API у Unisender **по умолчанию
недоступно** (антиспам-мера) — поэтому пока делаем это руками через ЛК
Unisender Go, по одному клиенту за раз. В коде это подключается через поле
`User.unisenderApiKey` — заполняет админ в `/app/admin` (форма «Unisender
Project» в таблице клиентов).

## Разовая настройка аккаунта (один раз, не на клиента)

1. Написать в поддержку Unisender Go с просьбой включить доступ к разделу
   «Настройки → Projects» и к API-методам работы с проектами. Без этого шага
   раздел Projects в личном кабинете не появится вообще.

## На каждого нового клиента

1. **Личный кабинет Unisender Go → Настройки → Projects → «Создать проект».**
   Назвать проект понятно (например, email клиента или название компании) —
   это единственное место, где потом можно опознать, какой проект чей.
2. Переключиться в контекст созданного проекта (селектор проекта в ЛК).
3. **Настройки → Домены отправки → добавить домен** — домен клиента (тот же,
   что клиент указывает в Smailee на `/app/sender`).
4. Unisender покажет DNS-записи, которые нужно прописать (SPF, DKIM,
   verification-TXT, опционально DMARC — см. общую инструкцию
   `https://godocs.unisender.ru/dns-setup`). Передать их клиенту, либо
   прописать самостоятельно, если домен у вас в управлении.
5. Подождать распространения DNS (от 30 минут до нескольких часов), нажать
   «Проверить» в ЛК Unisender — статус должен стать зелёным.
6. В настройках этого проекта найти его персональный **API-ключ** (отдельный
   от ключа основного аккаунта) и скопировать.
7. В Smailee: `/app/admin` → найти клиента в таблице → в колонке «Unisender
   Project» вставить скопированный ключ → «OK».
8. Клиент как обычно добавляет отправителя на `/app/sender` (email/имя/домен)
   и жмёт «Проверить DNS» — теперь эта кнопка реально дёргает Unisender API
   (`domain-validate-dkim`, `domain-validate-verification-record`) с ключом
   его проекта, а не мокает успех.

## Если понадобится автоматизировать позже

- Методы `project-create`/`project-update`/`project-list`/`project-delete`
  существуют и вызываемы через API — но только после того, как поддержка
  включит доступ к ним (шаг «Разовая настройка аккаунта» выше). Тогда шаги
  1–2 можно будет заменить вызовом `project-create` при регистрации клиента.
- Добавление самого домена (шаг 3) у Unisender **нет метода API** в принципе
  (`domain-list`/`domain-delete`/`domain-get-dns-records`/`domain-validate-*`
  есть, `domain-create` — нет) — этот шаг останется ручным всегда, независимо
  от Project.
